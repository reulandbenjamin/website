Station de mesure environnementale

Automatisation de collecte de données climatiques pour analyse scientifique

Contexte

Dans le cadre de mes études en électronique à la HEPL, j'ai développé une station de mesure autonome capable de collecter des données environnementales (température, humidité, pression, qualité de l'air) pour des applications de recherche.

Problème

Les stations météorologiques commerciales sont coûteuses et peu flexibles. L'objectif était de créer une solution open-source, modulaire et économique pour des mesures continues.

Solution

Utilisation d'un Raspberry Pi Zero W couplé à un HAT Enviro+ (Pimoroni) intégrant 7 capteurs :

  • BME280 : température, pression, humidité
  • LTR-559 : luminosité et proximité
  • MICS6814 : gaz (NO2, NH3, oxydants)
  • PMS5003 : particules fines (PM1.0, PM2.5, PM10)

Stack technique : Python 3, bibliothèque enviroplus, stockage CSV local, scripts de visualisation avec Matplotlib.

Impact

  • Données collectées toutes les 10 minutes pendant 3 mois
  • Coût total : ~60€ (vs 300€+ pour solutions commerciales)
  • Projet réutilisé par 2 collègues étudiants pour leurs TP
  • Publication prévue du code sur GitHub

Technologies

Python 3 Raspberry Pi Enviro+ HAT Matplotlib CSV

Apprentissages

  • Communication I²C et UART (niveau débutant)
  • Gestion de l'alimentation pour autonomie
  • Traitement de données temporelles
  • Documentation technique
